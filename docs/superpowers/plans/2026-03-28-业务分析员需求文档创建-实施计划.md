# 非功能需求文档

## 1. 性能需求
- **整体响应时间**: API响应<200ms，页面加载<2s，登录接口<100ms
- **并发支持**: 支持10,000+并发用户，峰值50,000并发
- **数据规模**: 支持1000万+注册用户，100万+活跃用户
- **吞吐量**: 登录模块支持10,000 TPS
- **可用性**: 99.9%可用性（月度停机<43分钟）
- **部门管理性能**:
  - 部门树查询: 响应时间 < 100ms（10万部门规模）
  - 部门结构调整: 响应时间 < 500ms
  - 部门成员查询: 响应时间 < 200ms（1万成员规模）
  - 部门数据同步: 支持1000+并发同步操作

## 2. 安全需求
- **认证**: JWT + OAuth2.0，支持 SSO
- **密码**: BCrypt 加密，支持密码策略配置
- **审计**: 100% 敏感操作可追溯
- **传输**: HTTPS/TLS 1.3
- **会话**: 自动超时，支持会话管理
- **防护**: 防暴力破解（账户锁定 + 限流）
- **部门安全**:
  - **部门权限隔离**: 不同部门的用户默认不能访问其他部门数据
  - **部门负责人授权**: 部门负责人只能管理本部门及下级部门
  - **部门变更审计**: 所有部门结构调整操作必须记录审计日志
  - **敏感部门保护**: 对财务、HR等敏感部门设置额外访问控制
  - **部门数据加密**: 部门间数据传输需加密，防止信息泄露

## 3. 技术架构需求
- **Redis使用**: 会话存储、限流控制、热点数据缓存、JWT黑名单
- **Kafka使用**: 异步日志写入、登录事件流、监控数据收集
- **负载均衡**: Nginx/ALB 多实例负载均衡
- **CDN**: 静态资源加速，减轻服务器压力
- **缓存策略**: 热点数据缓存，减轻数据库压力

## 4. 可用性与兼容性
- **浏览器支持**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **移动端**: 响应式设计，支持平板和手机
- **无障碍**: WCAG 2.1 Level AA
- **国际化**: 支持中英文切换

## 5. 扩展性需求
- **微服务**: 支持微服务架构拆分
- **多租户**: 预留多租户支持能力
- **水平扩展**: 支持无状态服务水平扩展

## 6. 维护与运维
- **监控告警**: 关键指标监控，异常自动告警
- **日志管理**: 结构化日志，支持集中收集
- **备份恢复**: 定期数据备份，灾难恢复预案
- **文档要求**: 完整的API文档和部署指南

# 部门管理模块需求文档

## 1. 模块概述
- **模块目的**: 管理企业组织架构的部门树形结构，支持多层级部门管理和部门成员管理
- **范围**: 包含部门CRUD、部门树形结构管理、部门成员管理、部门权限继承
- **主要用户**: 系统管理员、部门经理
- **业务价值**: 维护企业组织架构，支持基于部门的权限控制和数据隔离

## 2. 用户故事
- **US-004: 管理部门树形结构** (引用自USER_STORIES.md)
- **US-005: 管理部门成员** (引用自USER_STORIES.md)

## 3. 功能需求
### 3.1 部门树形结构管理
- **部门创建**: 支持创建部门，设置部门名称、代码、负责人、描述等信息
- **层级设置**: 支持设置部门层级关系，最多5级深度
- **路径维护**: 采用Materialized Path技术存储部门路径，支持快速查询子部门
- **可视化展示**: 前端提供可拖拽的树形结构可视化界面
- **批量操作**: 支持批量导入/导出部门结构

### 3.2 部门成员管理
- **成员查看**: 部门经理可以查看本部门及下级部门成员列表
- **成员调整**: 支持将用户调整到不同部门
- **负责人设置**: 可以设置部门负责人，负责人自动获得部门管理权限
- **权限继承**: 下级部门默认继承上级部门的权限配置

### 3.3 部门数据权限
- **数据范围**: 支持基于部门的数据权限控制（全部/本部门/下级部门/个人）
- **权限继承**: 上级部门的权限配置可自动继承到下级部门
- **权限覆盖**: 下级部门可以覆盖上级部门的权限设置

## 4. 验收标准
- [ ] AC1: 部门树查询响应时间 < 100ms（10万部门规模）
- [ ] AC2: 部门结构调整响应时间 < 500ms
- [ ] AC3: 支持最多5级部门层级
- [ ] AC4: 部门成员调整记录审计日志
- [ ] AC5: 部门树形结构可视化界面支持拖拽调整

## 5. 模块特有非功能需求
- **性能要求**:
  - 部门树查询: 响应时间 < 100ms（10万部门规模）
  - 部门结构调整: 响应时间 < 500ms
  - 部门成员查询: 响应时间 < 200ms（1万成员规模）
  - 部门数据同步: 支持1000+并发同步操作
- **安全要求**:
  - 部门权限隔离: 不同部门的用户默认不能访问其他部门数据
  - 部门负责人授权: 部门负责人只能管理本部门及下级部门
  - 部门变更审计: 所有部门结构调整操作必须记录审计日志
  - 敏感部门保护: 对财务、HR等敏感部门设置额外访问控制
- **可用性**: 部门树形界面直观易懂，支持拖拽操作
- **数据一致性**: 部门结构调整时自动更新所有子部门路径

## 6. 依赖关系
- **依赖模块**: 用户管理（获取用户信息）、角色权限管理（部门权限配置）
- **外部依赖**: HR系统（组织架构同步）、LDAP/AD集成

```python
#!/usr/bin/env python3
"""
需求文档验证脚本
验证所有需求文档的完整性、一致性和格式正确性
"""

import os
import re
import sys
from pathlib import Path

def check_file_exists(filepath):
    """检查文件是否存在"""
    if not os.path.exists(filepath):
        print(f"❌ 文件不存在: {filepath}")
        return False
    print(f"✅ 文件存在: {filepath}")
    return True

def check_markdown_structure(filepath, expected_sections):
    """检查Markdown文档结构"""
    with open(filepath, 'r', encoding='utf-8') as f:
        content = f.read()

    # 检查主要章节
    sections = re.findall(r'^##\s+', content, re.MULTILINE)

    if len(sections) >= expected_sections:
        print(f"✅ {filepath}: 包含{len(sections)}个主要章节（期望至少{expected_sections}个）")
        return True
    else:
        print(f"❌ {filepath}: 只包含{len(sections)}个主要章节（期望至少{expected_sections}个）")
        return False

def check_user_stories_references():
    """检查用户故事引用一致性"""
    print("\n🔍 检查用户故事引用一致性...")

    # 检查USER_STORIES.md包含所有用户故事
    with open('USER_STORIES.md', 'r', encoding='utf-8') as f:
        us_content = f.read()

    us_stories = re.findall(r'US-\d{3}', us_content)
    print(f"USER_STORIES.md包含{len(us_stories)}个用户故事: {', '.join(sorted(set(us_stories)))}")

    # 检查每个模块文档是否正确引用用户故事
    modules = {
        'USER_MANAGEMENT.md': ['US-001', 'US-002', 'US-003'],
        'DEPARTMENT_MANAGEMENT.md': ['US-004', 'US-005'],
        'ROLE_PERMISSION_MANAGEMENT.md': ['US-006', 'US-007', 'US-008'],
        'AUDIT_LOG.md': ['US-009', 'US-010'],
        'AUTHENTICATION_AUTHORIZATION.md': ['US-011', 'US-012']
    }

    all_passed = True
    for module_file, expected_stories in modules.items():
        filepath = f"prompts/requirements/{module_file}"
        if os.path.exists(filepath):
            with open(filepath, 'r', encoding='utf-8') as f:
                content = f.read()

            missing_stories = []
            for story in expected_stories:
                if story not in content:
                    missing_stories.append(story)

            if not missing_stories:
                print(f"✅ {module_file}: 正确引用所有期望的用户故事")
            else:
                print(f"❌ {module_file}: 缺少用户故事引用: {', '.join(missing_stories)}")
                all_passed = False
        else:
            print(f"⚠️  文件不存在: {filepath}")
            all_passed = False

    return all_passed

def check_nfr_coverage():
    """检查非功能需求覆盖"""
    print("\n🔍 检查非功能需求覆盖...")

    with open('NON_FUNCTIONAL_REQUIREMENTS.md', 'r', encoding='utf-8') as f:
        nfr_content = f.read()

    # 检查NFR文档的主要章节
    nfr_sections = ['性能需求', '安全需求', '技术架构需求', '可用性与兼容性', '扩展性需求', '维护与运维']

    all_passed = True
    for section in nfr_sections:
        if f"## {section}" in nfr_content:
            print(f"✅ NON_FUNCTIONAL_REQUIREMENTS.md包含: {section}")
        else:
            print(f"❌ NON_FUNCTIONAL_REQUIREMENTS.md缺少: {section}")
            all_passed = False

    return all_passed

def main():
    """主验证函数"""
    print("🔍 开始需求文档验证...")

    # 检查所有必需文件是否存在
    required_files = [
        'NON_FUNCTIONAL_REQUIREMENTS.md',
        'USER_STORIES.md',
        'prompts/requirements/USER_MANAGEMENT.md',
        'prompts/requirements/DEPARTMENT_MANAGEMENT.md',
        'prompts/requirements/ROLE_PERMISSION_MANAGEMENT.md',
        'prompts/requirements/AUDIT_LOG.md',
        'prompts/requirements/AUTHENTICATION_AUTHORIZATION.md',
        'prompts/requirements/SYSTEM_CONFIGURATION.md'
    ]

    files_exist = all(check_file_exists(f) for f in required_files)

    # 检查文档结构
    print("\n🔍 检查文档结构...")
    structure_checks = [
        ('NON_FUNCTIONAL_REQUIREMENTS.md', 6),
        ('USER_STORIES.md', 2),
        ('prompts/requirements/USER_MANAGEMENT.md', 6),
        ('prompts/requirements/DEPARTMENT_MANAGEMENT.md', 6),
        ('prompts/requirements/ROLE_PERMISSION_MANAGEMENT.md', 6),
        ('prompts/requirements/AUDIT_LOG.md', 6),
        ('prompts/requirements/AUTHENTICATION_AUTHORIZATION.md', 6),
        ('prompts/requirements/SYSTEM_CONFIGURATION.md', 6)
    ]

    structure_passed = all(check_markdown_structure(f, s) for f, s in structure_checks)

    # 检查一致性
    user_stories_passed = check_user_stories_references()
    nfr_passed = check_nfr_coverage()

    # 总结
    print("\n📊 验证总结:")
    print(f"文件存在检查: {'✅ 通过' if files_exist else '❌ 失败'}")
    print(f"文档结构检查: {'✅ 通过' if structure_passed else '❌ 失败'}")
    print(f"用户故事引用: {'✅ 通过' if user_stories_passed else '❌ 失败'}")
    print(f"非功能需求覆盖: {'✅ 通过' if nfr_passed else '❌ 失败'}")

    all_passed = files_exist and structure_passed and user_stories_passed and nfr_passed

    if all_passed:
        print("\n🎉 所有验证通过！需求文档创建完成。")
        return 0
    else:
        print("\n❌ 验证失败，请检查上述问题。")
        return 1

if __name__ == '__main__':
    sys.exit(main())
```
