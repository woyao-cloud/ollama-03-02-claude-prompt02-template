# ADR-010: 系统配置管理设计

## 状态
Accepted

## 背景
根据FRD v1.2需求，系统需要支持动态配置管理，包括邮件服务配置、安全策略配置、性能配置等。需要设计一个灵活、可扩展的配置管理系统，支持运行时配置更新、版本管理、审计追踪等功能。

## 决策
采用 **数据库存储 + 内存缓存 + 动态刷新** 的配置管理方案，支持多类型配置、加密存储、版本管理和审计追踪。

## 架构设计

### 1. 配置分类

| 配置类型 | 示例配置 | 特点 | 更新频率 |
|----------|----------|------|----------|
| **邮件配置** | SMTP服务器、端口、认证信息 | 敏感信息需要加密 | 低 |
| **安全策略** | 密码策略、登录策略、会话策略 | 影响系统安全行为 | 中 |
| **性能配置** | 缓存TTL、连接池参数、接口阈值 | 影响系统性能 | 中 |
| **系统参数** | 公司信息、Logo、默认设置 | 业务相关配置 | 低 |
| **功能开关** | 特性开关、实验性功能 | 控制功能可用性 | 高 |

### 2. 数据模型设计

```sql
-- 系统配置表
CREATE TABLE system_config (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    config_key VARCHAR(100) UNIQUE NOT NULL,
    config_value TEXT,
    config_type VARCHAR(50) NOT NULL,  -- EMAIL/SECURITY/PERFORMANCE/SYSTEM/FEATURE
    data_type VARCHAR(20) NOT NULL,    -- STRING/NUMBER/BOOLEAN/JSON
    description VARCHAR(500),
    is_encrypted BOOLEAN DEFAULT FALSE,
    is_sensitive BOOLEAN DEFAULT FALSE,
    default_value TEXT,
    min_value TEXT,
    max_value TEXT,
    regex_pattern VARCHAR(200),
    options JSONB,  -- 可选值列表，用于下拉选择
    version INT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_by UUID REFERENCES user(id),
    CONSTRAINT chk_config_type CHECK (config_type IN (
        'EMAIL', 'SECURITY', 'PERFORMANCE', 'SYSTEM', 'FEATURE'
    )),
    CONSTRAINT chk_data_type CHECK (data_type IN (
        'STRING', 'NUMBER', 'BOOLEAN', 'JSON', 'ARRAY'
    ))
);

-- 配置变更历史表
CREATE TABLE config_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    config_id UUID NOT NULL REFERENCES system_config(id),
    old_value TEXT,
    new_value TEXT,
    change_type VARCHAR(20) NOT NULL,  -- CREATE/UPDATE/DELETE
    changed_by UUID REFERENCES user(id),
    changed_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    reason VARCHAR(500)
);

-- 邮件模板表
CREATE TABLE email_template (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    template_code VARCHAR(50) UNIQUE NOT NULL,
    template_name VARCHAR(100) NOT NULL,
    subject VARCHAR(200) NOT NULL,
    content TEXT NOT NULL,
    variables JSONB,  -- 模板变量定义，如 {username}, {activationLink}
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 具体实现

### 1. 配置服务核心类

```java
@Service
@Slf4j
public class ConfigService {

    @Autowired
    private ConfigRepository configRepository;

    @Autowired
    private ConfigHistoryRepository historyRepository;

    @Autowired
    private CacheManager cacheManager;

    @Autowired
    private EncryptionService encryptionService;

    /**
     * 获取配置值（带缓存）
     */
    @Cacheable(value = "systemConfig", key = "#configKey")
    public String getConfigValue(String configKey) {
        SystemConfig config = configRepository.findByConfigKey(configKey)
            .orElseGet(() -> getDefaultConfig(configKey));

        String value = config.getConfigValue();

        // 解密加密的配置
        if (config.isEncrypted() && value != null) {
            try {
                value = encryptionService.decrypt(value);
            } catch (EncryptionException e) {
                log.error("配置解密失败: {}", configKey, e);
                throw new ConfigDecryptionException(configKey, e);
            }
        }

        return value;
    }

    /**
     * 更新配置
     */
    @CacheEvict(value = "systemConfig", key = "#configKey")
    @Transactional
    public void updateConfig(UpdateConfigRequest request, UUID userId) {
        SystemConfig config = configRepository.findByConfigKey(request.getConfigKey())
            .orElseThrow(() -> new ConfigNotFoundException(request.getConfigKey()));

        // 验证配置值
        validateConfigValue(config, request.getValue());

        // 保存历史记录
        saveConfigHistory(config, request.getValue(), userId, request.getReason());

        // 更新配置
        String newValue = request.getValue();
        if (config.isEncrypted() && newValue != null) {
            newValue = encryptionService.encrypt(newValue);
        }

        config.setConfigValue(newValue);
        config.setUpdatedBy(userId);
        config.setVersion(config.getVersion() + 1);
        configRepository.save(config);

        // 发布配置变更事件
        publishConfigChangedEvent(config, request.getValue());
    }

    /**
     * 批量获取配置
     */
    @Cacheable(value = "systemConfigs", key = "#configType")
    public Map<String, String> getConfigsByType(ConfigType configType) {
        return configRepository.findByConfigType(configType)
            .stream()
            .collect(Collectors.toMap(
                SystemConfig::getConfigKey,
                config -> {
                    String value = config.getConfigValue();
                    if (config.isEncrypted() && value != null) {
                        return encryptionService.decrypt(value);
                    }
                    return value;
                }
            ));
    }
}
```

### 2. 配置验证器

```java
@Component
public class ConfigValidator {

    public void validateConfigValue(SystemConfig config, String value) {
        if (value == null && config.isRequired()) {
            throw new ConfigValidationException(
                config.getConfigKey(), "配置值不能为空");
        }

        // 根据数据类型验证
        switch (config.getDataType()) {
            case NUMBER:
                validateNumber(config, value);
                break;
            case BOOLEAN:
                validateBoolean(config, value);
                break;
            case JSON:
                validateJson(config, value);
                break;
            case ARRAY:
                validateArray(config, value);
                break;
            default:
                validateString(config, value);
        }

        // 验证范围
        if (config.getMinValue() != null && config.getMaxValue() != null) {
            validateRange(config, value);
        }

        // 验证正则表达式
        if (config.getRegexPattern() != null) {
            validateRegex(config, value);
        }

        // 验证可选值
        if (config.getOptions() != null && !config.getOptions().isEmpty()) {
            validateOptions(config, value);
        }
    }

    private void validateNumber(SystemConfig config, String value) {
        try {
            Double.parseDouble(value);
        } catch (NumberFormatException e) {
            throw new ConfigValidationException(
                config.getConfigKey(), "必须为数字");
        }
    }
}
```

### 3. 动态配置应用

#### 3.1 邮件配置动态应用

```java
@Configuration
@ConfigurationProperties(prefix = "mail")
@RefreshScope
public class MailConfig {

    private String host;
    private Integer port;
    private String username;
    private String password;
    private String protocol;
    private Map<String, String> properties = new HashMap<>();

    @Bean
    @RefreshScope
    public JavaMailSender mailSender() {
        JavaMailSenderImpl mailSender = new JavaMailSenderImpl();
        mailSender.setHost(host);
        mailSender.setPort(port);
        mailSender.setUsername(username);
        mailSender.setPassword(password);
        mailSender.setProtocol(protocol);
        mailSender.setJavaMailProperties(createMailProperties());
        return mailSender;
    }

    private Properties createMailProperties() {
        Properties props = new Properties();
        properties.forEach(props::setProperty);
        return props;
    }
}
```

#### 3.2 安全策略动态应用

```java
@Configuration
@ConfigurationProperties(prefix = "security.policy")
@RefreshScope
public class SecurityPolicyConfig {

    private PasswordPolicy passwordPolicy;
    private LoginPolicy loginPolicy;
    private SessionPolicy sessionPolicy;

    @Bean
    @RefreshScope
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(passwordPolicy.getStrength());
    }

    @Bean
    @RefreshScope
    public AuthenticationManager authenticationManager(
            UserDetailsService userDetailsService,
            PasswordEncoder passwordEncoder) {

        DaoAuthenticationProvider provider = new DaoAuthenticationProvider();
        provider.setUserDetailsService(userDetailsService);
        provider.setPasswordEncoder(passwordEncoder);
        provider.setHideUserNotFoundExceptions(false);

        // 应用登录策略
        if (loginPolicy.isPreventConcurrentLogin()) {
            provider.setPreAuthenticationChecks(
                new ConcurrentLoginCheck(userDetailsService, sessionPolicy));
        }

        return new ProviderManager(Collections.singletonList(provider));
    }
}
```

### 4. 配置变更监听器

```java
@Component
public class ConfigChangeListener {

    @EventListener
    public void handleConfigChanged(ConfigChangedEvent event) {
        log.info("配置变更: {} = {}", event.getConfigKey(), event.getNewValue());

        // 清除相关缓存
        cacheManager.evict("systemConfig", event.getConfigKey());

        // 根据配置类型执行特定操作
        switch (event.getConfigType()) {
            case SECURITY:
                refreshSecurityContext();
                break;
            case PERFORMANCE:
                refreshCacheConfig();
                break;
            case EMAIL:
                refreshMailConfig();
                break;
        }

        // 发送通知
        sendConfigChangeNotification(event);
    }

    private void refreshSecurityContext() {
        // 重新加载安全配置
        SecurityContextHolder.clearContext();
    }
}
```

## 性能优化

### 1. 多级缓存策略

```java
@Configuration
public class CacheConfig {

    @Bean
    public CacheManager cacheManager() {
        CaffeineCacheManager cacheManager = new CaffeineCacheManager();

        // 配置缓存策略
        cacheManager.setCaffeine(Caffeine.newBuilder()
            .expireAfterWrite(5, TimeUnit.MINUTES)  // 配置缓存5分钟
            .maximumSize(1000)
            .recordStats());

        return cacheManager;
    }

    @Bean
    public RedisCacheManager redisCacheManager(RedisConnectionFactory factory) {
        RedisCacheConfiguration config = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(10))  // Redis缓存10分钟
            .serializeValuesWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new Jackson2JsonRedisSerializer<>(Object.class)));

        return RedisCacheManager.builder(factory)
            .cacheDefaults(config)
            .build();
    }
}
```

### 2. 配置预加载

```java
@Component
public class ConfigPreloader implements ApplicationRunner {

    @Autowired
    private ConfigService configService;

    @Override
    public void run(ApplicationArguments args) {
        // 预加载关键配置到缓存
        List<String> criticalConfigs = Arrays.asList(
            "security.password.minLength",
            "security.login.maxAttempts",
            "mail.smtp.host",
            "cache.user.ttl"
        );

        criticalConfigs.forEach(configService::getConfigValue);
        log.info("预加载 {} 个关键配置到缓存", criticalConfigs.size());
    }
}
```

## 后果

### 正面影响
1. **灵活性高**：支持运行时配置更新，无需重启服务
2. **安全性好**：敏感配置加密存储，审计追踪完整
3. **可维护性强**：配置集中管理，版本控制，历史可查
4. **性能优秀**：多级缓存，配置读取快速

### 负面影响
1. **实现复杂度**：需要处理配置验证、加密、缓存一致性等
2. **数据库依赖**：配置存储在数据库，增加数据库压力
3. **缓存一致性**：配置更新后需要及时刷新缓存

### 缓解措施
1. **简化设计**：MVP版本先实现核心功能，逐步增强
2. **性能优化**：使用缓存，批量读取，异步更新
3. **监控告警**：监控配置查询性能，缓存命中率

## 替代方案

### 方案A：配置文件（application.yml）
```yaml
mail:
  host: smtp.example.com
  port: 587
security:
  password:
    minLength: 8
```
**放弃原因**：无法运行时更新，不支持UI管理，无版本控制。

### 方案B：配置中心（Nacos/Apollo）
```java
@NacosValue(value = "${mail.host}", autoRefreshed = true)
private String mailHost;
```
**放弃原因**：引入外部依赖，增加系统复杂度，学习成本高。

### 方案C：环境变量
```bash
export MAIL_HOST=smtp.example.com
export MAIL_PORT=587
```
**放弃原因**：管理不便，无UI界面，不适合复杂配置。
