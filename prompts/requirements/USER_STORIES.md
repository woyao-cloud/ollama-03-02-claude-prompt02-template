## 按模块分类的用户故事

### 用户管理模块

#### US-001: 创建用户
- **作为** 系统管理员
- **我希望** 创建新用户账号
- **以便** 新员工可以访问系统
- **验收标准**:
  - [ ] AC1: 可以填写用户基本信息（姓名、邮箱、部门）
  - [ ] AC2: 系统发送激活邮件到新用户邮箱
  - [ ] AC3: 可以为新用户分配初始角色
  - [ ] AC4: 支持批量导入用户（Excel/CSV）
  - [ ] AC5: 创建成功后显示确认信息

#### US-002: 用户自助注册
- **作为** 新员工
- **我希望** 自助注册账号
- **以便** 快速开始使用系统
- **验收标准**:
  - [ ] AC1: 可以通过邮箱注册
  - [ ] AC2: 邮箱需要验证
  - [ ] AC3: 密码需符合安全策略（8位以上，包含大小写和数字）
  - [ ] AC4: 注册后进入待审批状态
  - [ ] AC5: 注册成功后收到欢迎邮件

#### US-003: 用户资料管理
- **作为** 普通用户
- **我希望** 管理我的个人资料
- **以便** 保持信息最新
- **验收标准**:
  - [ ] AC1: 可以修改姓名、头像、联系方式
  - [ ] AC2: 可以修改密码（需验证旧密码）
  - [ ] AC3: 可以启用双因素认证
  - [ ] AC4: 可以查看登录历史
  - [ ] AC5: 修改敏感信息后收到通知邮件

### 部门管理模块

#### US-004: 管理部门树形结构
- **作为** 系统管理员
- **我希望** 管理部门的树形结构
- **以便** 维护企业组织架构
- **验收标准**:
  - [ ] AC1: 可以创建、编辑、删除部门
  - [ ] AC2: 可以设置部门层级关系（最多5级）
  - [ ] AC3: 可以拖拽调整部门排序
  - [ ] AC4: 部门变更时自动更新所有子部门路径
  - [ ] AC5: 部门树形结构可视化展示

#### US-005: 管理部门成员
- **作为** 部门经理
- **我希望** 管理部门成员
- **以便** 了解团队组成
- **验收标准**:
  - [ ] AC1: 可以查看部门成员列表
  - [ ] AC2: 可以调整用户所属部门
  - [ ] AC3: 可以设置部门负责人
  - [ ] AC4: 可以查看部门层级（上级和下级部门）
  - [ ] AC5: 部门成员变动记录审计日志

### 角色权限管理模块

#### US-006: 创建角色
- **作为** 系统管理员
- **我希望** 创建自定义角色
- **以便** 匹配组织架构
- **验收标准**:
  - [ ] AC1: 可以定义角色名称和描述
  - [ ] AC2: 可以为角色分配权限（菜单、按钮、数据范围）
  - [ ] AC3: 可以设置角色的数据可见范围（全部/部门/个人）
  - [ ] AC4: 角色可以继承其他角色的权限
  - [ ] AC5: 创建的角色立即可用

#### US-007: 分配用户角色
- **作为** 系统管理员
- **我希望** 为用户分配角色
- **以便** 控制用户权限
- **验收标准**:
  - [ ] AC1: 可以为单个用户分配多个角色
  - [ ] AC2: 可以批量为用户分配角色
  - [ ] AC3: 可以查看用户的所有权限（包括继承的）
  - [ ] AC4: 角色变更实时生效（或需重新登录）
  - [ ] AC5: 角色分配有审计日志

#### US-008: 细粒度权限控制
- **作为** 系统管理员
- **我希望** 配置细粒度权限
- **以便** 精确控制访问
- **验收标准**:
  - [ ] AC1: 支持菜单级权限（显示/隐藏）
  - [ ] AC2: 支持操作级权限（增删改查）
  - [ ] AC3: 支持字段级权限（显示/编辑）
  - [ ] AC4: 支持数据级权限（范围控制）
  - [ ] AC5: 权限变更实时生效

### 审计日志模块

#### US-009: 查看操作日志
- **作为** 系统管理员
- **我希望** 查看用户操作日志
- **以便** 审计和追溯
- **验收标准**:
  - [ ] AC1: 可以按用户、时间、操作类型筛选日志
  - [ ] AC2: 日志包含用户、IP、时间、操作、结果
  - [ ] AC3: 支持导出日志为 Excel/PDF
  - [ ] AC4: 支持日志保留策略配置
  - [ ] AC5: 关键操作（登录、权限变更）有实时告警

#### US-010: 个人操作历史
- **作为** 普通用户
- **我希望** 查看自己的操作历史
- **以便** 了解账号活动
- **验收标准**:
  - [ ] AC1: 可以查看最近 30 天的登录记录
  - [ ] AC2: 可以查看权限变更历史
  - [ ] AC3: 发现异常活动可以报告
  - [ ] AC4: 支持导出个人活动报告

### 认证授权模块

#### US-011: 密码策略管理
- **作为** 系统管理员
- **我希望** 配置密码策略
- **以便** 提升账号安全
- **验收标准**:
  - [ ] AC1: 可以设置密码复杂度要求
  - [ ] AC2: 可以设置密码过期时间
  - [ ] AC3: 可以设置历史密码限制（不能重复使用）
  - [ ] AC4: 可以配置账户锁定策略（失败次数）
  - [ ] AC5: 策略变更对所有用户生效

#### US-012: 双因素认证
- **作为** 普通用户
- **我希望** 启用双因素认证
- **以便** 提升账号安全
- **验收标准**:
  - [ ] AC1: 支持 TOTP（Google Authenticator）
  - [ ] AC2: 支持短信验证码
  - [ ] AC3: 支持邮箱验证码
  - [ ] AC4: 可以下载恢复码
  - [ ] AC5: 管理员可以强制要求 2FA
